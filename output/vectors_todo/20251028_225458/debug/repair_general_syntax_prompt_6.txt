## Operators
Verus extends Rust logical operators with low-precedence forms that are especially helpful in specification code:

Standard Operators: &&, ||, ==>, <==>
Low-Precedence Variants: &&& and |||

The meaning of &&& is the same as && (logical AND), and ||| is the same as || (logical OR), but with lower precedence. This allows you to write conditions in a "bulleted list" style that remains grouped in a logical manner:

```
&&& a ==> b
&&& c
&&& d <==> e && f
```

is equivalent to:

```
(a ==> b) && c && (d <==> (e && f))
```

Note:
- Implication (==>) and equivalence (<==>) bind more tightly than &&& and |||.
- Using &&&/||| can make long specifications clearer by grouping logical clauses neatly.

fn binary_search(v: &Vec<u64>, k: u64) -> (r: usize)

        forall|i: int, j: int| 0 <= i && i <= j < v@.len() ==> v@[i] <= v@[j],

/// Reverses the elements of a vector in place.
///
/// # Arguments
/// * `v` - A mutable reference to a vector of u64 integers to be reversed
fn reverse(v: &mut Vec<u64>)

    ensures
        v@.len() == old(v)@.len(),
        forall|i: int|

            v@[i] == old(v)@[old(v)@.len() - i - 1]
{
    let length = v.len();
    let ghost v_old = v@;
    for n in 0..(length / 2)
        invariant
            length == v.len(),
            // For indices below n, they are swapped with their mirror
            forall|i: int|
                0 <= i && i < n ==>
                v@[i] == v_old[length as int - i - 1]

    {
        let x = v[n];
        let y = v[length - 1 - n];
        v.set(n, y);
        v.set(length - 1 - n, x);
    }
}
